## Context

IAQ 用 ViT 注意力给 patch 分配量化比特，重要 patch 多给 bit。要让收端按这套变长分配
反量化，必须知道每 patch 的比特深度（M_i 图）。把 M_i 过信道（创新点 A）一错就游标
失步、级联崩到随机猜（"元信息悬崖"）。前序 change `iaq-a1-metadata-free`（A1，收端从
基础层派生分配）已在实验中证伪：

- A1 派生需基础层**逐 bit 精确**才能两端一致，而任何有限 FEC 都给不了这个保证；
- A1 的增强层仍是变长游标 → 基础层错 1 bit 即级联崩（只是被基础层兜在底，非随机）；
- 用 KL 蒸馏干净图注意力去"修派生"的微调（§2.5）反而把注意力摊平、准确率倒退 7pt
  （代理目标与真目标背离）。

诚实结论：**变长 + 免传 + 抗误码三者在裸游标下不可兼得。** 本 change 改 AI-centric
思路解决之。零成本探针（`scripts/run_bucket_probe.py`，n=256）已验证关键假设（见 Decisions）。

约束（项目级）：中文交流与注释；本机 GPU 满载会过热，训练在另一台 GPU 机跑、本机仅
CPU 小样本；接受训练但不做 ImageNet 级（CIFAR-100 规模 OK）；新颖度求"不雷同"非首创。

## Goals / Non-Goals

**Goals:**
- 保住 IAQ 的动态重要性分配（底色），同时让信道差错只造成**局部损伤、绝不级联崩**。
- 用一个**收端学习式语义修补网络**（唯一创新内核）把局部损伤补回，优雅退化、无悬崖。
- 六线、同总信道比特、双场景（匹配/失配）对照，把 AI/派生范式相对经典 FEC 的优势
  （失配鲁棒 + 零额外带宽的 AI 增益）讲清楚。

**Non-Goals:**
- 不追求在**匹配 SNR** 下超过经典"FEC 保护元信息"（元信息仅占载荷 0.27%，经典修法在
  匹配场景就近乎完美；本方案立足点在失配 + 范式，不自欺）。
- 不做 ImageNet 级训练；不宣称跨数据集泛化（除非另测）。
- 不实现注水法分配、不引入 LDPC（FEC 用重复码即可）。
- 不保留 A1 的派生增强层机制（已证伪，留作反面对照与动机）。

## Decisions

### D1 源编码：均匀基础层 + 固定桶重要性增强层（非级联）
- **基础层**：每 patch m_base bit 均匀量化，**定长、无需排序**，破解"要排序先重建、
  要重建先知分配"的死循环；同时是粗图先验 + 派生排序的来源。
- **增强层**：按收端派生的注意力排序，把 patch 装进**固定大小、固定比特**的桶
  （例 b392：基础层 1bit×196；增强 top-49 +3bit / 次-49 +1bit / 其余 98 +0；
  增强码流 `[49×3][49×1]` 定长）。总 ΣM_i=392 = ρ0.25。
- **为何不级联**：增强层按固定位置读，永远对齐；派生排序若与发端不一致，只是把值
  贴错几个 patch（**局部损伤**），不可能游标雪崩。
- **替代方案**：A1 变长游标（已证级联崩，弃）；纯均匀（丢重要性，弃）；定长满槽
  （退化≈均匀，弃）。固定桶是"保重要性 + 非级联"的最小公倍数。
- **探针证据**（n=256）：b392 下 增量(理想)0.613 / 均匀0.395 / 粗桶0.539 →
  **粗桶捞回 66% 重要性红利**；据基础层派生分桶 vs 据原图，**仅掉 2pt**，错配
  集中在桶边界（严重跨档仅 14%）→ 派生误差局部、良性，正是 AI 可修对象。

### D2 收端派生排序（metadata-free，但错误局部化）
收端解基础层 → 粗图 → 算注意力 → 排序入桶。这步的产物就是"比特深度图"，**收端
解码必然已算出**（解码要用），零额外开销。派生与发端不一致只导致局部错配（D1）。

### D3 学习式语义修补网络（创新核心）
- **结构**：轻量 patch-Transformer（几层自注意力），**像素/patch 域**输出残差修正
  （可视化友好、冻结分类器原样复用、好训）；本质 MAE/inpainting。
- **输入**：受损重建 patch 网格 + **比特深度=可靠性图**（D2 的免费副产品，作为可靠性
  通道，告诉网络每个 patch 该信几分）。重要性在此**二次进场指导修复**，不再分配比特。
- **兜底**：输出残差加回原重建 → 最坏≈不修≈粗图地板，**结构保证 AI 不帮倒忙**。
- **训练**：冻结分类裁判 + patch_embed，**只训修补器**；数据 = CIFAR-100 过完整管线
  （粗桶编码→AWGN 随机 SNR→解码）；**损失 = 分类交叉熵(主) + 重建 MSE(辅)**，
  跨 SNR 训。⚠️ **直接用任务准确率当目标**，不重蹈 §2.5 代理目标摊平的覆辙。
- **替代方案**：纯 token-embedding 域（更抽象难调）；显式通信校验定位（沾通信开销）；
  单独训异常检测头（多部件、错则连累）。选"像素域 + 免费可靠性图 + 隐式修复"。

### D4 传统通信仅辅助、基础层 FEC 可选
非级联使基础层差错只造成局部错配（D1）→ 基础层**无需逐 bit 精确** → FEC 从结构必需
降为**可选的"抬低 SNR 地板"旋钮**。主实验默认不加 FEC（最干净的同带宽）；附加对照
单列"加轻 FEC 抬地板"并按 gauge B 记开销。

### D5 实验：六线、同总信道比特、双场景
- 六线：⓪理想(oracle 天花板) / ①均匀(metadata-free 基线) / ③裸传元信息(悬崖) /
  ⑥IAQ+保护元信息(经典 FEC 修法) / ④粗桶(无修补) / ⑤粗桶+AI修补(本方案)。
- **同带宽记账**：全线总信道比特 = 768×392；③⑥ 把元信息(+FEC)**从预算内扣**
  （ΣM_i 让位 ~1–3 单位，对准确率可忽略但堵死"偷用 bit"质疑）；⓪ 与 ③ 同信源预算，
  唯一差别是元信息过不过信道 → **⓪−③ 隔离纯悬崖**；④−① 隔离重要性红利；
  **⑤−④ 隔离 AI 增益（零额外带宽）**。
- **双场景**：匹配（FEC 按当前 SNR 配，诚实展示 ⑥≈⓪ 经典也能修）；
  **失配（杀招，FEC 按固定高 SNR 配再扫真实 SNR）→ ⑥ 再悬崖、⑤ 平滑** = 核心卖点。
- **消融**：AI 修补开/关(⑤vs④)；可靠性图开/关；粗桶+修补 vs 均匀+修补（重要性是否仍值）；
  跨 SNR 训 vs 单点训。
- **指标**：Top-1 vs Eb/N0；AI 增益 ⑤−④；优雅性(无悬崖)；修补前后可视化；开销记账。

## Risks / Trade-offs

- [粗桶比理想 IAQ 粗，只捞回 ~66% 红利] → 接受："鲁棒+可部署"换"一点理想性"；低码率
  下红利大(b392 +14pt)，故工作点取 b392/ρ0.25。
- [匹配场景经典 FEC(⑥)≈理想，⑤ 未必赢] → 立足点明确为失配 + 范式，主战场设为失配场景，
  匹配场景如实呈现不藏。
- [极低 SNR 大部分 patch 损坏、上下文耗尽，AI 修不动] → 残差兜底优雅退回粗图地板，
  不崩；如实声明 AI 发力区为中等损伤。
- [b392 基础层只能 1bit，派生比探针(2bit)差] → 实现期用 m_base=1 重测钉死真实代价，
  由 AI 修补吸收；必要时调桶档位。
- [修补训练用代理目标会重蹈 §2.5 覆辙] → 明确用分类损失(任务目标)训练，残差兜底防退化。
- [训练吃 GPU、本机过热] → 训练在另一台 GPU 机跑，本机仅 CPU 小样本验证，先问后跑。

## Open Questions

- 修补网络规模/层数与训练 SNR 区间的最优配置，实现期调。
