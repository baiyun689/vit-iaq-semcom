## ADDED Requirements

### Requirement: 均匀基础层（定长、派生来源）
系统 SHALL 提供一个均匀基础层：每 patch 固定 `m_base` bit 均匀量化，码流**定长、位置固定**，
解码不依赖任何排序或元信息。基础层 MUST 同时充当（a）粗图先验、（b）收端派生排序的来源。

#### Scenario: 基础层解码不依赖排序
- **WHEN** 收端收到基础层码流
- **THEN** 系统按固定位置（每 patch `m_base` bit）解出全部 patch 的粗重建，无需任何排序或段长信息

#### Scenario: 基础层受损不级联
- **WHEN** 基础层若干 bit 被信道翻转
- **THEN** 仅对应 patch 的粗值变化（局部），其余 patch 解码不受影响，不发生游标失步

### Requirement: 固定桶重要性增强层
系统 SHALL 提供按重要性排序入桶的增强层：给定一组**固定大小、固定比特**的桶
（如 top-K1 加 b1 bit、次-K2 加 b2 bit、其余加 0），按注意力降序把 patch 填入桶。
增强码流 MUST 为**定长、固定位置**，每 patch 总比特 = `m_base + 桶增量`，总 ΣM_i 守恒于预算。

#### Scenario: 增强层定长定位读取
- **WHEN** 收端读取增强层码流
- **THEN** 系统按固定位置切出"K1 个 b1-bit 值、K2 个 b2-bit 值…"，对齐永不依赖派生分配是否正确

#### Scenario: 嵌入式重组
- **WHEN** 某 patch 基础层 `m_base` bit 与其桶增量 `d` bit 重组为 `m_base+d` bit 索引
- **THEN** 因均匀量化嵌入式性质，高 `m_base` 位=基础层、低 `d` 位=增强层，反量化得 `m_base+d` bit 精度

### Requirement: 收端派生排序入桶
系统 SHALL 在收端用基础层重建图算注意力、排序、入桶，得到每 patch 的桶号/比特深度，
**不传输任何排序或 M_i 元信息**。该比特深度图 MUST 作为解码副产品供下游（修补网络）使用。

#### Scenario: 派生不一致只造成局部错配
- **WHEN** 收端派生排序与发端不一致（基础层受损或浮点差异导致边界 patch 易桶）
- **THEN** 仅边界处少数 patch 拿到错配的细节值（局部损伤），码流结构对齐不变、不级联崩

#### Scenario: 比特深度图可供下游
- **WHEN** 收端完成派生入桶与解码
- **THEN** 系统输出每 patch 的比特深度（14×14 图），供修补网络作可靠性输入，无额外传输或重复计算
