## Why

IAQ 的动态重要性比特分配高效，但其变长解析对差错极脆弱：分配/元信息一旦出错，
收端游标失步 → 后续全部错位 → 灾难崩盘（创新点 A 的"元信息悬崖"）。前序方案
A1（收端从基础层派生分配）已被证伪：派生需基础层逐 bit 精确（物理上无法保证），
其变长增强层照样级联崩。本 change 换 AI-centric 思路——**不阉割动态分配来躲悬崖，
而是用"非级联源编码 + 收端学习式语义修补"把悬崖治好**：源编码层让差错只造成局部
损伤（绝不级联），AI 修补网络再把局部损伤补回来。传统通信只做轻量辅助，修补网络
是唯一创新内核。

## What Changes

- 新增**非级联粗桶源编码**：均匀基础层（定长，派生排序 + 粗图先验来源）+ 固定大小
  固定比特的重要性"桶"增强层（按收端派生排序入桶）。任何分配/信道差错只造成
  **局部值错误**，结构上不可能游标级联。
- 新增**收端学习式语义修补网络**（创新核心）：轻量 patch-Transformer，吃受损重建图
  + 免费的"比特深度=可靠性"图，用粗图先验 + 存活上下文修复局部损伤，残差兜底
  保证永不跌穿粗图地板；训练用分类损失（吸取 A1 用代理目标摊平注意力的教训）。
- `IAQPipeline` 增加 `bucket`（无修补）与 `bucket+repair`（本方案）模式分支。
- 新增六线对照实验（同总信道比特口径）：⓪理想 / ①均匀 / ③裸传元信息(悬崖) /
  ⑥IAQ+保护元信息(经典FEC修法) / ④粗桶 / ⑤粗桶+AI修补；双场景（匹配 / 失配杀招）。
- 公平性：全线同总信道比特；③⑥ 的元信息/FEC 从预算内扣（ΣM_i 让位），粗桶基础层
  FEC 降为**可选的抬地板旋钮**（非级联使基础层无需逐 bit 精确）。

## Capabilities

### New Capabilities
- `iaq-bucket-coding`: 非级联粗桶源编码 —— 均匀基础层 + 固定桶重要性增强层 +
  收端从基础层派生排序入桶；差错局部化、结构上不级联。
- `iaq-semantic-repair`: 收端学习式语义修补网络 —— patch-Transformer，输入受损重建
  + 比特深度可靠性图，残差修复局部损伤，分类损失训练、跨 SNR、残差兜底。

### Modified Capabilities
- `iaq-pipeline`: 增加 `bucket` / `bucket+repair` 模式分支与六线（双场景）对照评估。

## Impact

- 新代码：`src/vit_iaq_semcom/bucket.py`（粗桶编解码 + 派生排序）、
  `src/vit_iaq_semcom/repair.py`（修补网络 + 训练/推理）；扩展 `pipeline.py`。
- 新脚本：粗桶可行性探针已存（`scripts/run_bucket_probe.py`）；新增修补训练脚本与
  六线实验脚本。
- 依赖：复用既有 ViT 编码器（冻结分类裁判）、量化、信道（AWGN/BPSK）、重复码 FEC。
- 训练：修补网络小、单 GPU、CIFAR-100 train 可训（本机仅 CPU 小样本验证，GPU 另机跑）。
- 不破坏既有 `iaq` 模式（默认行为不变，向后兼容）。
