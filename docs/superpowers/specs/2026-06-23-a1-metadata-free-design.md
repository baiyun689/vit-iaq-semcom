# A1 元信息-free IAQ:设计文档

- 日期:2026-06-23
- 状态:设计稿(待实现)
- 关联:第二创新点(Plan B);接续创新点 A「元信息性能悬崖」(已验证)
- 工作点:b_target=392 bit(ρ=0.25),AWGN+BPSK,CIFAR-100 微调 ViT,top-1 准确率

## 1. 背景与动机

### 1.1 创新点 A 已确立的事实(悬崖)

IAQ 语义通信把 ViT 注意力转成 patch 重要性 a_i → 比特分配 M_i → 逐 patch 均匀量化
→ 数字信道 → 重建 → 分类。论文默认 M_i 分配图与 u_min/u_max **无损传输**。

创新点 A 的三线实验(`scripts/run_awgn_metadata_experiment.py`,n=256 高保真)证实:
一旦把元信息也过信道,低 SNR 下会**灾难性失步**——M_i 图错 1 bit → 增强载荷按错长度
解析 → 整图崩到 ~0.01。表现为准确率随 SNR 下降的**垂直悬崖**(③ 线 ≤7dB 反输给均匀 ① 线)。

工作点 b392 实测(base_acc=0.9336,n=256):
- 高 SNR(12dB):② IAQ 理想 0.613 vs ① 均匀 0.398,收益 ②−① ≈ +21.5pt
- ③ 裸传:≤4dB 崩到 ~0.01~0.03,~7dB 才追平 ①,10dB 起收敛到 ②

### 1.2 悬崖是「脆弱性」不是「开销」

元信息 B_add = ⌈log₂(M_max+1)⌉·N + 8·2 = 4·196+16 = 800 bit,相对载荷 ~30 万 bit
(768×392)仅占 **0.27%**。即使强 FEC(重复码 r=5)保护也才 ~1.3% 带宽。

**推论:经典 UEP 花极小带宽就能填平悬崖,在设计 SNR 点准确率 ≈ ②。** 因此本方案的
价值**不在开销**,而在结构性鲁棒与信道失配下的表现(见 §5)。

### 1.3 为什么需要 A1(AI 视角)

给元信息加 FEC/UEP 是经典信道编码,不构成 AI 创新,本文将其降为**对照基线**。
真正的创新点是**根本不传 M_i 图**:收发两端各自从一个共享的「基础层」重新派生出
**逐 bit 相同**的分配。失步机理(传错长度)从此不存在 → 悬崖被结构性消除。

## 2. 设计目标与非目标

**目标**
- 在 b392/ρ=0.25 同预算下,把③的垂直悬崖变成**有底斜坡**:高 SNR 达 ②,SNR 下降时平滑
  退化,极低 SNR 最坏退化到**基础层(1-bit)质量**,**绝不出现③式失步崩盘(跌到随机猜)**。
  诚实声明:该底可能低于 2-bit 均匀 ①——分层把半数比特押在增强层的代价;A1 价值在消除
  灾难崩盘与失配鲁棒,不在底部绝对值。
- 在**信道失配**场景(按固定 SNR 设计保护、真实 SNR 更差)下,UEP 会再悬崖而 A1 不崩。
- v1 **零训练**先跑通;训练(CIFAR-100 级,非 ImageNet 级)仅作可选增强。

**非目标**
- 不追求在设计 SNR 点的准确率优势(此处 A1 与 UEP 预期打平)。
- 不追求开销优势(A1 与 UEP 在开销上半斤八两,见 §4.3)。
- 不上 LDPC/Turbo;FEC 砌块只用重复码(速率 1/r,BER 有闭式)。
- 不做 ImageNet 级训练。

## 3. A1 数据流

### 3.1 两层切分(被预算锁死)

b392 = 平均每 patch 2 bit。两层都要非空,只有一种切法:

```
基础层 base    : 每 patch 固定 M_base = 1 bit 均匀量化  → 196 bit
增强层 enhance : 剩余预算按派生分配 ΔM_i 增量分配       → 196 bit
每 patch 总比特 = 1 (base) + ΔM_i (enhance)
```

- **基础层双用**:① 一张最粗的全图轮廓;② 两端各自喂进 ViT 算注意力 → 派生分配的唯一依据。
- 基础层是命门,加**轻 FEC**(重复码速率 1/r,r 待扫)保证两端逐 bit 一致恢复。

### 3.2 派生分配(两端各自、确定性一致)

```
发端 & 收端,各自独立执行:
  1. 解出基础层重建图 x_base(收端解 FEC 后得到与发端逐 bit 相同的 base)
  2. x_base 过共享 ViT → class-token→patch 注意力 a_i (196 个)
  3. 量化 a_i 到 16 档(4 bit)         ← 见 §3.3,锁死浮点漂移
  4. 增量比特分配算法(预算 196 bit,M_max_enh 待定)→ ΔM_i
发端:按 ΔM_i 取每 patch 增强比特,拼成增强层比特流
收端:按自己派生的 ΔM_i 切增强层比特流 → 反量化 → 叠加到 base
```

**关键:增强层每段多长,收端无需被告知。** base 逐 bit 无误 → 两端 ΔM_i 完全相同 →
增强层边界**自动对齐**。这就是 A1 删掉脆弱元信息的机制——没有「传错长度」这个失步源。

### 3.3 注意力量化锁一致性

两端 ViT 浮点前向可能有微小漂移(0.01230 vs 0.01231),直接喂分配算法可能让某 patch
两端排序翻转 → ΔM_i 错位。对策:把每个 a_i **量化到 16 档**再喂增量算法。微小漂移落进
同一粗档 → 两端拿到逐 bit 相同的「量化热力图」→ 分配确定性一致。

风险登记:本方案**假设**漂移大到需要量化才能对齐。实现前用一个纯 CPU 最小脚本实测
「同权重同输入跑两遍 attention_scores 是否逐 bit 一致」,据此确认量化档位(16 档为起点)。

### 3.4 同步标记(v1 不做)

base 逐 bit 无误时增强层边界自动对齐,无需同步标记。同步标记仅作「base 残余误码导致
派生偏移」的兜底,降为 v2 增强项,v1 不实现。

### 3.5 可选修复头(v1 不做)

若 1 bit 基础层算注意力质量不足(见 §6),退路之一是训一个小修复头(CIFAR-100 级)。
v1 零训练,不实现;作为 §6 风险实验后的可选项。

## 4. 对照实验

### 4.1 六条线(同 ρ=0.25)

| 线 | 含义 |
|---|---|
| ① 均匀 | uniform 分配,无重要性元信息,参照系 |
| ② IAQ 理想 | 元信息无损上界(不可实现,仅作天花板) |
| ③ 裸传 | 元信息过信道不保护 = 已做的悬崖 |
| ④ EEP | 元信息 & 载荷等强 FEC |
| ⑤ UEP | 元信息强 FEC、载荷弱 FEC |
| ⑥ A1 | 本方案,不传元信息 |

### 4.2 两张图

- **图1 匹配场景**:每个 SNR 按当前 SNR 设计保护强度 → ⑤≈⑥≈② 收敛,立「都能修」。
- **图2 失配场景(杀招)**:按固定点(如 8dB)设计 FEC,再扫真实 SNR。真实<设计时
  ⑤ UEP 元信息 FEC 失效 → **再悬崖**;⑥ A1 分配与信道无关 → **平滑有底**。A1 完胜。

### 4.3 开销对齐(预算口径 B:同信道带宽)

六条线对齐**同实传比特数**(同信道带宽),FEC 膨胀计入总预算,谁加 FEC 谁从信源比特挤。
理由:只有同带宽,图2 失配对决才公平——不让任何一方靠多占带宽虚胜。
诚实声明:A1 **不省带宽**;A1 的「基础层强制均匀」带来分配自由度损失,与 UEP 那
~1% 元信息 FEC 开销半斤八两。开销不是卖点,失配鲁棒性才是。

对照开销账(同带宽下):

| | 信源侧代价 | 信道侧代价 |
|---|---|---|
| UEP ⑤ | 多传 800 bit 元信息 | 给 800 bit 加强 FEC |
| A1 ⑥ | 基础层均匀→分配自由度损失(软、小) | 仅基础层轻 FEC,无 800 bit 元信息 |

## 5. 预期结论(自圆其说的论点)

1. **结构性免疫**:A1 不传 M_i 图 → 失步机理不存在 → 无灾难长尾,曲线有底(≥①)。
2. **信道无关**:派生分配只依赖 base,不依赖信道质量估计 → 失配下不崩(图2)。
3. **无独立受保护控制信道**:不需要为元信息单开一条强保护链路。

设计点准确率 A1≈UEP≈②(诚实,不夸大);区别全在低 SNR 与失配区。

## 6. 头号技术风险及验证实验

**风险:1 bit 基础层那张极粗轮廓图,过 ViT 算出的注意力,是否够接近原图注意力?**

- 够 → 派生分配 ≈ 理想分配,A1 零训练即成立。
- 不够 → 派生分配次优,准确率达不到 ②。退路:
  (i) 换工作点 b588/ρ=0.375 让 base 升到 2 bit(仍在论文扫描区间);
  (ii) 加小修复头(CIFAR-100 级训练);
  (iii) base 算注意力时用原图注意力蒸馏。

**验证实验(实现第一步,GPU 跑前先与用户约定时长/样本)**:
- 取一批图,分别用「原图」与「1 bit base 重建图」算 a_i;
- 度量两者派生 M_i 分配的一致性(如 Kendall τ / 排序重叠 / 最终准确率差);
- 据此判定走零训练还是某条退路。
